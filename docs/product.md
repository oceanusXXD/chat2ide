# 产品与场景

`chat2ide` 是一个自托管、单用户的 Codex CLI 远程终端工作台。

它把服务器上的 Codex CLI 会话保持为真实 PTY 进程，再通过浏览器或手机提供多标签终端、重连和最近输出回放。目标用户是有一台可信 Linux 开发机、VPS 或家用服务器的个人开发者。

## 核心场景

你在服务器上的项目目录里运行 Codex CLI，让它修 bug、写测试或整理文档。任务可能持续很久，也可能需要你随时接管。`chat2ide` 让你不必每次从手机 SSH、配密钥、处理断线或忍受小屏终端输入，而是打开一个受 PIN 保护的网页入口继续操作。

## 用户故事

1. 我在电脑上启动 Codex 修复问题，离开电脑后用手机继续看输出。
2. 我同时开多个 Codex 会话，分别处理 bug、测试和文档。
3. 我刷新页面或网络断开后，能重新附着到还在运行的终端并看到最近输出。
4. 我通过 Cloudflare Tunnel 暴露一个 HTTPS 入口，不直接开放服务器端口。
5. 我用专门的低权限系统账户运行服务，把 `CODEX_CWD` 限制在目标项目目录。

## 产品边界

`chat2ide` 只服务单用户、单服务器场景。它不是多人协作 IDE、不是远程桌面、不是容器沙箱，也不是审计平台。

登录后的用户拥有运行 `chat2ide` 的系统账户权限。任何能在 Codex CLI 或 shell 中执行的命令，都应按真实服务器命令来理解。

## 设计取舍

- 终端输出保留为 PTY 字节流，不重组为聊天消息。
- 使用 `xterm.js` 显示终端，尽量保留 ANSI、光标控制和交互式行为。
- 使用内存保存 session、终端进程和 ring buffer，不引入数据库。
- Cloudflare Tunnel 是推荐公网入口，应用默认监听 `127.0.0.1`。
- 新建终端后，首次 WebSocket attach 才启动 PTY，避免启动阶段输出在浏览器尚未准备好时丢失。

## 成功标准

一个完整可用部署应该满足：

- 能从公网 HTTPS 域名打开 PIN 登录页。
- 登录后能创建多个 Codex CLI 终端。
- 手机端能通过底部输入栏发送提示词或命令。
- 刷新页面后能恢复仍在内存中的终端并回放最近输出。
- `/api/health` 正常返回。
- `npm run preflight` 能证明服务器关键前置条件可用。
- 用户清楚知道服务重启会丢失当前终端和输出缓存。
