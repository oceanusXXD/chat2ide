# 产品和使用场景

`chat2ide` 是一个自托管、单用户的 Codex CLI 远程终端工作台。

它把服务器上的 Codex CLI 会话保持为真实 PTY 进程，再通过浏览器或手机提供多标签终端、重连和最近输出回放。目标用户是有一台可信 Linux 开发机、VPS 或家用服务器的个人开发者。

## 用户画像

### 独立开发者

这类用户通常已经把 Codex CLI、SSH、项目依赖和密钥配置在一台长期在线的 Linux 机器上。他们不需要多人 IDE，只需要在离开电脑后继续看任务输出，必要时从手机发一条命令或中断当前进程。

### 小型开源维护者

维护者常常同时处理 issue、测试失败、文档和发布前检查。`chat2ide` 可以把这些任务放在不同终端标签里。维护者可以在桌面上开始任务，再用手机查看结果或发送下一步指令。

这类用户需要快速判断哪条后台任务有新输出、哪条终端异常退出，以及关闭一个任务后应继续看相邻任务，而不是回到随机标签。

### 自托管工具用户

这类用户关心部署边界和依赖数量。`chat2ide` 不要求数据库，也不带多用户账号系统。推荐部署方式是 Cloudflare Tunnel 加 `127.0.0.1` 本地监听，并用低权限系统账户运行。

## 主要场景

你在服务器上的项目目录里运行 Codex CLI，让它修 bug、写测试或整理文档。任务可能持续很久，也可能需要你随时接管。`chat2ide` 让你不必每次从手机 SSH、配密钥、处理断线或忍受小屏终端输入，而是打开一个受 PIN 保护的网页入口继续操作。

## 使用场景

### 接管正在运行的 Codex CLI

用户登录后可以创建终端、看到真实 PTY 输出、发送输入，并在刷新页面后重新附着到仍在内存里的终端。服务重启会丢失登录状态、PTY 进程和输出缓存。

### 从手机查看长任务

手机端首屏保留连接状态、当前终端、工作台概览、终端输出和底部输入栏。常用操作包括发送、`Ctrl+C`、停止、重启、关闭和刷新输出。页面不需要横向滚动。

底部输入栏保存本次浏览器会话内最近 30 条已发送内容，方便从手机重复 `npm test`、`git status`、`codex` 提示词等短命令。历史只在内存中，不写入浏览器本地存储。

### 通过 Cloudflare Tunnel 暴露入口

服务监听 `127.0.0.1`，由 Cloudflare Tunnel 转发公网 HTTPS 域名。生产环境使用 `APP_PIN_HASH`、`APP_PUBLIC_ORIGIN`、登录失败限速、WebSocket origin 校验、资源上限和低权限系统账户。

### 作为开源项目维护

仓库提供 README、配置说明、架构说明、协议文档、运维手册、安全边界、贡献指南、测试入口和 preflight。贡献者先读文档就能判断项目边界。

## 用户故事

1. 我在电脑上启动 Codex 修复问题，离开电脑后用手机继续看输出。
2. 我同时开多个 Codex 会话，分别处理 bug、测试和文档。
3. 我刷新页面或网络断开后，能重新附着到还在运行的终端并看到最近输出。
4. 我通过 Cloudflare Tunnel 暴露一个 HTTPS 入口，不直接开放服务器端口。
5. 我用专门的低权限系统账户运行服务，把 `CODEX_CWD` 限制在目标项目目录。

## 产品边界

`chat2ide` 只服务单用户、单服务器场景。它不是多人协作 IDE、远程桌面、容器沙箱或审计平台。

登录后的用户拥有运行 `chat2ide` 的系统账户权限。任何能在 Codex CLI 或 shell 中执行的命令，都应按真实服务器命令来理解。

## 设计取舍

- 终端输出保留为 PTY 字节流，不重组为聊天消息。
- 使用 `xterm.js` 显示终端，保留 ANSI、光标控制和交互式行为。
- 使用内存保存 session、终端进程和 ring buffer，不引入数据库。
- Cloudflare Tunnel 是推荐公网入口，应用默认监听 `127.0.0.1`。
- 新建终端后，首次 WebSocket attach 才启动 PTY，避免启动阶段输出在浏览器尚未准备好时丢失。
- 终端列表按创建时间稳定排序；关闭当前标签后优先切到相邻任务，符合多终端连续处理的工作流。
- 命令历史只做浏览器内存缓存，提升移动端重复输入效率，同时避免持久化敏感命令。
- 默认限制终端数量、单次输入大小和 WebSocket 消息大小。这个项目不是无限 PTY 托管器。

## 成功标准

一个可用部署需要满足：

- 能从公网 HTTPS 域名打开 PIN 登录页。
- 登录后能创建多个 Codex CLI 终端。
- 手机端能通过底部输入栏发送提示词或命令。
- 多终端状态、后台未读输出和异常状态能在顶部概览中直接看到。
- 单行输入能从本次会话历史中回填已发送命令。
- 刷新页面后能恢复仍在内存中的终端并回放最近输出。
- `/api/health` 正常返回。
- `npm run preflight` 能证明服务器关键前置条件可用。
- 用户清楚知道服务重启会丢失当前终端和输出缓存。
