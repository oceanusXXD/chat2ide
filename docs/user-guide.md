# 使用指南

## 登录

打开部署后的网址，输入服务器上配置的 PIN。登录成功后，浏览器会保存 `HttpOnly` session cookie，并建立到 `/ws` 的 WebSocket 连接。

如果 session 过期，页面会回到 PIN 登录页。

## 新建终端

先在顶部“启动”选择一个 profile，再点击“新建终端”。默认 profile 来自 `CODEX_COMMAND` / `CODEX_ARGS` / `CODEX_CWD`，额外 profile 来自 `TERMINAL_PROFILES`。

每个 profile 会显示命令、工作目录和说明，便于在手机上确认将要启动的是 Codex CLI、其他 AI coding CLI、shell wrapper，还是 `codex --remote` 这类远程 app-server 入口。

实现上，HTTP API 会先创建一个 `starting` 状态的会话。浏览器 WebSocket 附着到该标签页后，服务端才真正启动 PTY 进程。这样启动阶段的交互式输出能进入真实 xterm 视图。

如果某个产品没有独立可运行 CLI，而是只有 IDE 插件、桌面客户端或本机 companion，就不要把它伪装成 profile。应该由那个客户端通过 `/bridge` 把自己的 session 发布给 `chat2ide`，这样它会以 `client_bridge` 后端出现在同一个终端列表里。

## 输入

桌面端可以直接聚焦终端输入，也可以使用底部输入栏。

手机端建议始终使用底部输入栏。输入命令或提示词后点击“发送”，内容会追加回车并写入当前运行中的会话：PTY 会写入服务器上的真实终端，Bridge 会转发给发布该 session 的客户端。

底部输入栏会在浏览器内存中保存本次会话最近 30 条已发送内容。单行输入时可以用方向键向上/向下回填历史；多行内容仍保持 textarea 的正常编辑行为。退出登录或刷新浏览器会清空这份本地历史。

## 多终端

每个标签页对应一个独立会话。PTY 标签页对应服务器上的真实 PTY 进程；Bridge 标签页对应 IDE 插件、桌面客户端或本机 companion 发布的外部 session。可以同时保留多个会话，例如：

- 一个会话修 bug。
- 一个会话跑测试。
- 一个会话整理文档或查看日志。

切换标签页时，浏览器会重新附着目标终端并回放最近输出。为了避免多后台高速输出压垮前端，浏览器同一时刻只订阅当前活动标签页的实时输出。

桌面端顶部概览会显示终端总数、当前 profile、运行/启动/停止/异常数量、后台未读输出和当前终端尺寸。手机端使用更紧凑的 session 状态栏，显示连接点、当前终端、profile、未读数和终端尺寸。关闭当前标签时，页面优先切到右侧相邻标签；如果右侧没有标签，则切到左侧标签。

## 控制按钮

`Ctrl+C`

向当前会话发送中断字符。PTY 会收到 `Ctrl+C`；Bridge 会把同一个输入字符转发给客户端，由客户端决定如何处理。

`停止`

PTY：结束当前进程，标签页保留，状态变为已停止。

Bridge：向客户端发送 `control: stop` 请求，是否停止由客户端确认并通过 `session_status` 回报。

`重启`

PTY：清空当前终端显示和 ring buffer，并启动一个新的 PTY。

Bridge：清空本地回放缓存并向客户端发送 `control: restart` 请求。

`关闭`

PTY：结束进程并移除标签页。

Bridge：向客户端发送 `control: close`，然后从移动端列表移除该 bridge session。

`刷新输出`

强制重新 attach 当前终端，并从服务端 ring buffer 回放最近输出。

`重连`

主动重建 WebSocket。适合弱网、手机切换网络后使用。

## 输出恢复

刷新浏览器页面或短暂断线后，只要服务端进程还在，已有终端仍在内存中。重新登录或自动恢复 session 后，页面会列出这些终端并回放最近输出。

注意：ring buffer 不是完整日志。服务重启后，登录 session、终端进程和输出缓存都会消失。

bridge 会话也是同样的内存模型：客户端重新连接后可以继续 upsert 自己的 session，但如果服务重启，列表和回放都会消失。

## 手机端建议

- 使用底部输入栏输入，不依赖 xterm 原生键盘。
- 用顶部状态栏先判断连接状态、当前终端、未读数和终端尺寸。
- 横向滚动标签页切换终端。
- 弱网恢复后先点“重连”或刷新页面。
- 常用短命令可以用方向键从本次会话历史里回填，敏感命令不会写入浏览器本地存储。
- 长命令可以先在手机输入法里编辑完整，再一次发送。
- 小屏幕上不要同时开太多高速输出任务，必要时只保留当前关注的终端。
