# 使用指南

## 登录

打开部署后的网址，输入服务器上配置的 PIN。登录成功后，浏览器会保存 `HttpOnly` session cookie，并建立到 `/ws` 的 WebSocket 连接。

如果 session 过期，页面会回到 PIN 登录页。

## 新建终端

点击“新建终端”会创建一个 Codex CLI 终端标签页。

实现上，HTTP API 会先创建一个 `starting` 状态的会话。浏览器 WebSocket 附着到该标签页后，服务端才真正启动 PTY 进程。这样启动阶段的交互式输出能进入真实 xterm 视图。

## 输入

桌面端可以直接聚焦终端输入，也可以使用底部输入栏。

手机端建议始终使用底部输入栏。输入命令或提示词后点击“发送”，内容会追加回车并写入当前 Codex CLI PTY。

底部输入栏会在浏览器内存中保存本次会话最近 30 条已发送内容。单行输入时可以用方向键向上/向下回填历史；多行内容仍保持 textarea 的正常编辑行为。退出登录或刷新浏览器会清空这份本地历史。

## 多终端

每个标签页对应一个独立 PTY 进程。可以同时保留多个 Codex CLI 会话，例如：

- 一个会话修 bug。
- 一个会话跑测试。
- 一个会话整理文档或查看日志。

切换标签页时，浏览器会重新附着目标终端并回放最近输出。为了避免多后台高速输出压垮前端，浏览器同一时刻只订阅当前活动标签页的实时输出。

顶部概览会显示终端总数、运行/启动/停止/异常数量、后台未读输出和当前终端尺寸。关闭当前标签时，页面优先切到右侧相邻标签；如果右侧没有标签，则切到左侧标签。

## 控制按钮

`Ctrl+C`

向当前 PTY 发送中断字符，适合停止正在运行的命令。

`停止`

结束当前 PTY 进程，标签页保留，状态变为已停止。

`重启`

清空当前终端显示和 ring buffer，并启动一个新的 PTY。

`关闭`

结束进程并移除标签页。

`刷新输出`

强制重新 attach 当前终端，并从服务端 ring buffer 回放最近输出。

`重连`

主动重建 WebSocket。适合弱网、手机切换网络后使用。

## 输出恢复

刷新浏览器页面或短暂断线后，只要服务端进程还在，已有终端仍在内存中。重新登录或自动恢复 session 后，页面会列出这些终端并回放最近输出。

注意：ring buffer 不是完整日志。服务重启后，登录 session、终端进程和输出缓存都会消失。

## 手机端建议

- 使用底部输入栏输入，不依赖 xterm 原生键盘。
- 用顶部概览先判断是否有后台未读输出、异常终端或仍在启动的会话。
- 横向滚动标签页切换终端。
- 弱网恢复后先点“重连”或刷新页面。
- 常用短命令可以用方向键从本次会话历史里回填，敏感命令不会写入浏览器本地存储。
- 长命令可以先在手机输入法里编辑完整，再一次发送。
- 小屏幕上不要同时开太多高速输出任务，必要时只保留当前关注的终端。
